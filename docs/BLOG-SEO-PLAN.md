# Blog + SEO Plan — Move & Meditate

> Authoritative plan for the blog section and organic search strategy. Living doc — update as the strategy evolves.

## Goal

Build a 10-article cornerstone blog under `/blog` that:
1. Ranks for high-intent local + informational queries Amisha's audience actually searches
2. Funnels readers to the four pillar pages (`/dance`, `/yoga`, `/weddings`, `/corporate`) as the conversion surface
3. Establishes E-E-A-T signals (Amisha as the named expert) so AI Overviews + Google trust the brand
4. Hits a 6-month milestone of ~1,500 organic clicks/month with page-1 rankings for at least three Bangalore-local terms

## Audience + scope

- **Where**: India, primarily Bangalore. International only as a side benefit.
- **Language**: English.
- **Intent mix**: ~50% local-commercial (Bangalore queries), ~30% informational top-funnel (yoga nidra, breathwork), ~20% transactional (sangeet choreography, couple dance).
- **Voice**: Amisha's first-person where natural. Warm, specific, never AI-generic. Real practice details (instructor pet-peeves, what beginners get wrong) beat keyword-stuffing every time.
- **Conversion path**: read article → click pillar link → fill contact form OR tap WhatsApp invite.

## Keyword strategy

### Keyword research findings (May 2026)

Live research confirmed several things before picking topics:

- **91.5% of Indians believe yoga aids in prevention / management of health conditions** — vast top-of-funnel demand exists.
- **35% of Indian adults have practised yoga at some point** — broad addressable audience.
- **Stated motivations among Indian practitioners**: 61% strength + flexibility, 59% sleep quality + mindfulness, 56% reducing burnout + mental stress. → "Yoga for sleep", "yoga for stress", "yoga for back pain" cluster has both proven demand AND research backing.
- **Government tailwind**: India's Ministry of Ayush rolled out clinical yoga protocols targeting lifestyle diseases (diabetes, hypertension) in 2026. AI Overviews and Google now treat .gov.in yoga sources as authoritative → linking to them in articles boosts trust.
- **Long-tail dominates**: 91.8% of all Google searches are long-tail (3+ words). Target keyword difficulty < 30 — that's where a small new domain can rank within months, not years.
- **"is yoga good for…" People-Also-Ask cluster** is enormous in India: back pain, weight loss, knees, anxiety, PCOS, posture, sleep. Every one of those is a citable, low-competition long-tail.
- **Sangeet 2026 specifics**: the format has shifted from "one big stage-show" to "curated mixtape with short sets and deliberate reels". Trending song picks editors love include Kesariya, Raatan Lambiyan, What Jhumka, Naach Punjaban, Gallan Goodiyan, Makhna, Desi Girl, Teri Ore. → A "Best Sangeet Songs 2026 + Choreography Notes" article picks up both `sangeet songs 2026` and `sangeet choreography` traffic in one piece.
- **Corporate wellness India 2026**: the 2026 conversation has moved past "yoga at lunch" to mental-health support, hybrid burnout, meeting hygiene, nutrition. Bangalore tech HR teams are actively budgeting for this.
- **Local commercial keywords ("yoga classes Bangalore", "dance classes Bangalore") have the highest competition** because old domains squat the head term. We'll likely rank for **neighborhood-modified variants** (Indiranagar, Koramangala, HSR, Whitefield) before the head term — that's normal and still drives qualified leads.

## Sanity schema: `blogPost`

```ts
{
  name: 'blogPost',
  fields: [
    title: string (req)
    slug: slug (req, source: title)
    excerpt: text (req, 60-160 chars — used for meta description + card)
    heroImage: image with alt + caption (req)
    author: reference → 'author' (req, defaults to Amisha)
    category: string list (dance | yoga | weddings | corporate) (req)
    tags: string[] (free-form, editor-defined)
    publishedAt: datetime (req — drives sort + dateline)
    updatedAt: datetime (optional — shown when set)
    readingTime: number (auto-calculated client-side, override-able)
    body: portable-text (req)
      // marks: bold, italic, link, internalLink (slug picker)
      // types: heading (h2/h3), figure (image + caption), callout, code, blockquote, ctaCard, table
    faq: array of { question, answer } (optional — drives FAQPage JSON-LD)
    relatedPillar: string (dance|yoga|weddings|corporate, optional — drives sidebar CTA)
    relatedPosts: reference[] → 'blogPost' (max 3, optional — overrides auto-recs)
    seo: {
      title: string (optional override; defaults to title)
      description: string (optional override; defaults to excerpt)
      ogImage: image (optional override; defaults to heroImage)
      noindex: boolean (default false)
    }
    published: boolean (default true)
  ],
  preview: { title, subtitle: category, media: heroImage }
}
```

Plus minimal `author` schema (Amisha singleton initially, expandable later):

```ts
{
  name: 'author',
  fields: [
    name, role, bio (text), photo (image), credentials (string[]),
    social: { instagram, youtube, linkedin },
  ]
}
```

## Routes + rendering

### `/blog` (index)
- Server component, fetches latest 12 posts via GROQ
- Pagination at 12 per page (cursor or `?page=` query)
- Filterable by category via tab strip
- Each card: hero image, category eyebrow, title, excerpt, reading time, date

### `/blog/[slug]`
- Server component, fetches single post via GROQ (slug match)
- `generateStaticParams` from CMS at build time → static prerender per article
- ISR with `revalidateTag('blog')` triggered by webhook on publish
- Sections: breadcrumb · hero image · title · meta row (author, date, reading time, category) · TOC (sticky on desktop) · article body · FAQ accordion · author bio card · related-pillar CTA · 3-post related-articles strip
- `Article` + `BlogPosting` + `FAQPage` (when applicable) + `BreadcrumbList` JSON-LD
- OG image: dynamic via `app/blog/[slug]/opengraph-image.tsx` (1200x630, wordmark + title + hero crop)

### `/blog/rss.xml`
- Route handler emitting RSS 2.0 + Atom-equivalent
- Last 30 published posts
- Helps AI crawlers (Perplexity, ChatGPT) discover content faster than sitemap-only

## SEO + metadata per post

Every article must hit these floors before publish:

- **Title tag**: 50-60 chars, primary keyword in first 40 chars, brand suffix
- **Meta description**: 140-160 chars, secondary keyword + clear value
- **H1**: exact-match or near-match primary keyword
- **First paragraph**: primary keyword in first 100 words
- **URL**: short, keyword-led slug (no dates, no IDs)
- **Heading structure**: 5-8 H2s, H3s under as needed. One H1 only.
- **Word count**: 1,800-2,500 for cornerstone, 1,200+ for support
- **Internal links**: minimum 3 to pillar pages, 2 to other blog posts
- **External links**: 2-4 to authoritative sources (PubMed for science claims, gov.in for regs, established practitioner sites — never to competitors)
- **Hero image alt**: descriptive, includes primary keyword naturally
- **At least one quotable one-sentence definition** in the first 200 words (AI Overview citation magnet)
- **FAQ section**: 4-6 questions answering the People-Also-Ask variants for the primary keyword
- **Author byline + bio card**: links to /author/amisha
- **Last updated date** shown prominently (Google rewards freshness signals)
